# 善-文言文省词元

> **言简意赅，一字千金。** Skill 自身以文言写成（107行），自证其效。

将 AI 回复从白话转为精简文言文，较普通白话节省 64% 字符量，较"简洁回答"额外节省 35%。**v2.0 新增上下文压缩**——压缩全对话为文言摘要，可贴至新对话续命。

## 核心创新

文言文信息密度远超白话——中文独有的 token 优化路径。技术内容（代码/报错/路径）原样保留，只压缩自然语言。

**v2.0 Skill 自身仅 107 行（~600 token），较 v1 省 80%。约 5 轮对话即可回本。**

## 效果对比

```
正常 AI（约 120 字符）：
"你好！我很乐意帮你解决这个 bug。根据你的描述，在第 42 行出现了
因为变量 user 为 null 而导致的错误。这是一个常见的空值引用问题..."

简洁中文（约 65 字符，省 46%）：
"第 42 行有 bug，user 为 null 导致错误。加空值检查即可。需要代码吗？"

文言 Full（约 35 字符，省 71%）：
"L42: user 或 null→TypeError。宜加空检：if(user){...}。需代码否？"
```

## 三档压缩

| 级别 | 压缩率 | 风格 | 适用 |
|------|--------|------|------|
| 🪶 Lite | 30-40% | 完整句法，去客套话 | 日常对话、新用户 |
| 🪨 Full | 50-65% | 碎片化，主语省略 | 技术讨论（默认） |
| 🔥 Ultra | 65-87% | 电报体，符号化 | 快速确认、老手交流 |

## 特性

- **Skill 自身文言写成**：107 行，~600 token，自身即为效果示范
- **词汇对照**：70+ 日常高频词 + 6 大领域专词
- **10 种句式转换**：祈使/疑问/条件/因果/让步/判断/描述/并列/比较/确认
- **技术内容保护**：代码/报错/路径/URL/命令一字不改
- **中英混合处理**：英文术语保留，只压中文描述
- **安全阀**：危险操作/复杂流程/用户困惑→自动白话
- **双语文风格**：《论语》体 + 史传体
- **上下文压缩**：`/善compact` 压缩全对话为文言摘要，可续新对话

## 快速开始

### SOLO
1. SOLO → Skills 管理 → 创建 Skill
2. 粘贴 `SKILL.md` 全文
3. 名称设为"善-文言文省词元"
4. 对话输入 `/善` 激活

### 其他 AI 工具
将 `SKILL.md` 粘贴到"自定义指令"/"系统提示词"即可。

## 命令

| 命令 | 功能 |
|------|------|
| `/善` | 激活，默认 Full |
| `/善 lite\|full\|ultra\|off` | 切换压缩级别 |
| `/善 style lunyu\|shizhuan` | 切换风格 |
| `/善commit` / `/善c` | 文言 Git 提交信息 |
| `/善review` / `/善r` | 文言代码审查 |
| `/善compact` / `/善z` | **压缩全对话为文言摘要** |

## 上下文压缩

长对话 token 膨胀？`/善compact` 将全对话压为 ≤300 字文言摘要：

```
# 对话摘要
## 任务
## 已做
## 关键代码/命令（原样）
## 待解
## 技术要点（原样）
```

摘要可贴至新对话首条，即续上下文。代码/报错/路径原样保留。

## 安全机制

危险操作（`DROP`/`rm -rf`/生产部署）→ 白话确认。用户说"说人话"→ 立即白话。

## 性能数据

基于 10 项常见开发任务，三组对比（字符数）：

| 任务 | 正常 | 简洁 | 文言 | vs正常 | vs简洁 |
|------|------|------|------|--------|--------|
| 解释 React 重渲染 | 180 | 95 | 42 | 77% | 56% |
| 修认证中间件 bug | 120 | 65 | 25 | 79% | 62% |
| 数据库连接池配置 | 350 | 170 | 68 | 81% | 60% |
| Docker 多阶段构建 | 170 | 85 | 52 | 69% | 39% |
| PostgreSQL 竞态调试 | 195 | 90 | 40 | 79% | 56% |
| React 错误边界 | 420 | 190 | 75 | 82% | 61% |
| 回调改 async/await | 65 | 45 | 45 | 31% | 0%* |
| PR 安全审查 | 105 | 58 | 58 | 45% | 0%* |
| git rebase vs merge | 110 | 60 | 48 | 56% | 20% |
| 微服务 vs 单体架构 | 75 | 48 | 48 | 36% | 0%* |
| **平均** | **179** | **91** | **50** | **64%** | **35%** |

> `*` 三项简洁中文已足够精炼。其余七项文言较简洁额外节省 48%。
