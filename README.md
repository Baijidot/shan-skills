# 善-文言文省词元

> **言简意赅，一字千金。**
> 将 AI 回复从白话转为精简文言文，较普通白话节省 64% 字符量，较"简洁回答"额外节省约 35%。专为中文开发者打造。

## 这是什么？

一个通用 AI Skill，安装后 AI 用**文言文风格**回复，大幅减少输出。

**核心创新：** 文言文信息密度远超白话——这是中文独有的优化路径。技术内容（代码、报错、路径）完整保留，只压缩自然语言部分。

## 效果对比

```
正常 AI（约 120 字符）：
"你好！我很乐意帮你解决这个 bug。根据你的描述，在第 42 行出现了
因为变量 user 为 null 而导致的错误。这是一个常见的空值引用问题..."

简洁中文（约 65 字符，省 46%）：
"第 42 行有 bug，user 为 null 导致错误。加空值检查即可。需要代码吗？"

文言文 Full（约 35 字符，省 71%）：
"L42: user 或 null→TypeError。宜加空检：if(user){...}。需代码否？"
```

> **额外收益：** 相比仅要求"简洁回答"，文言文模式再省约 25% 字符数。

## 三档压缩

| 级别 | 压缩率 | 风格 | 适用 |
|------|--------|------|------|
| 🪶 Lite | 30-40% | 完整句法，去客套话 | 日常对话、新用户 |
| 🪨 Full | 50-65% | 碎片化，主语省略 | 技术讨论（默认） |
| 🔥 Ultra | 65-87% | 电报体，符号化 | 快速确认、老手交流 |

## 特性

- **词汇对照**：70+ 日常高频词 + 6 大领域专用词汇（前端/后端/数据库/DevOps/测试/Git）
- **10 种句式转换**：祈使、疑问、条件、因果、让步、判断、描述、并列、比较、确认
- **技术内容保护**：代码、报错、路径、URL、命令绝对不改一字
- **中英混合处理**：保留代码标识符和技术缩写，全英文句子和代码片段原样保留
- **安全阀机制**：危险操作、复杂流程、用户困惑时自动切回白话
- **双语文风格**：《论语》体（极简）+ 史传体（典雅）
- **3 个子技能**：文言提交 / 文言审查 / 文言压缩

## 快速开始

### SOLO
1. 打开 SOLO → Skills 管理 → 创建 Skill
2. 将 `SKILL.md` 内容粘贴到 Skill 编辑区
3. 保存，名称设为"善-文言文省词元"
4. 对话中输入 `/善` 激活

### 其他 AI 工具
将 `SKILL.md` 内容粘贴到工具的"自定义指令"/"系统提示词"设置区即可。

## 命令

| 命令 | 功能 |
|------|------|
| `/善` | 激活，默认 Full 模式 |
| `/善 lite\|full\|ultra\|off` | 切换压缩级别 |
| `/善 style lunyu\|shizhuan` | 切换风格 |
| `/善commit` / `/善c` | 生成文言 Git 提交信息 |
| `/善review` / `/善r` | 文言代码审查评论 |
| `/善compress` / `/善z` | 压缩文件为文言文 |

## 安全机制

危险操作（删库、生产部署）自动切回白话确认。用户说"不明白"立刻恢复正常。

## 性能数据

基于 10 项常见开发任务，三组对比（字符数统计）：

| 任务 | 正常回复 | 简洁中文 | 文言 Full | 文言 vs 正常 | 文言 vs 简洁 |
|------|----------|----------|-----------|-------------|-------------|
| 解释 React 重渲染 | 180 | 95 | 42 | 77% | 56% |
| 修认证中间件 bug | 120 | 65 | 25 | 79% | 62% |
| 数据库连接池配置 | 350 | 170 | 68 | 81% | 60% |
| Docker 多阶段构建 | 170 | 85 | 52 | 69% | 39% |
| PostgreSQL 竞态调试 | 195 | 90 | 40 | 79% | 56% |
| React 错误边界 | 420 | 190 | 75 | 82% | 61% |
| 回调改 async/await | 65 | 45 | 45 | 31% | 0%* |
| PR 安全审查 | 105 | 58 | 58 | 45% | 0%* |
| git rebase vs merge | 110 | 60 | 48 | 56% | 20% |
| 微服务 vs 单体架构 | 75 | 48 | 48 | 36% | 0%* |
| **平均** | **179** | **91** | **50** | **64%** | **35%** |

> `*` 三项任务简洁中文已经足够精炼，文言未再进一步压缩。其余七项文言较简洁中文额外节省约 48%。

> 注：上表为字符数统计。实际 token 消耗因模型分词器而异。输出节省仅涉及可见文本，思考 token 不受影响。
